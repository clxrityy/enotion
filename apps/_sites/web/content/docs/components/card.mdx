## Import

```tsx

```

## Basic Usage

```tsx

  <h2>Card Title</h2>
  <p>This is some content inside the card.</p>

```

## Props

```typescript
interface CardProps extends HTMLAttributes<HTMLDivElement> {
  children: ReactNode;
  palette?: ColorPaletteType;
}
```

### children
- **Type:** **ReactNode**
- **Required:** Yes
- Content to display inside the card

### palette
- **Type:** **ColorPaletteType**
- **Default:** **undefined**
- Color palette for styling

## CSS Custom Properties

When using a palette, the component sets these CSS variables:

- **--card-box-shadow** - Default box shadow
- **--card-hover-box-shadow** - Box shadow on hover
- **--card-backdrop-background-color** - Backdrop color
- **--card-hover-background-color** - Background on hover
- **--card-active-background-color** - Background when active
- **--card-hover-border-color** - Border color on hover
- **--card-active-border-color** - Border color when active
- **--card-border-color** - Default border color
- **--card-backdrop-border-color** - Backdrop border color
- **--card-backdrop-box-shadow** - Backdrop shadow
- **--card-backdrop-hover-box-shadow** - Backdrop shadow on hover

## Styling

The Card component uses the **enotion-card** CSS class and can be customized through:

1. **Color Palettes**: Apply pre-defined color schemes
2. **Custom Classes**: Add your own CSS classes
3. **Inline Styles**: Override specific style properties
4. **CSS Variables**: Customize the component's CSS custom properties

```css
.my-custom-card {
  border-radius: 16px;
  padding: 2rem;
}

.my-custom-card:hover {
  transform: translateY(-4px);
  transition: transform 0.2s;
}
```

## Accessibility

The Card component:
- Uses semantic **\<div>** element
- Supports all standard HTML div attributes
- Can be made interactive with proper ARIA attributes when needed
- Inherits accessibility features from provided props
