## Props

```typescript
interface ButtonProps extends ComponentPropsWithoutRef<"button"> {
  children: ReactNode;
  palette?: ColorPaletteType;
  variant?:
    | "default"
    | "outline"
    | "primary"
    | "secondary"
    | "tertiary"
    | "info"
    | "success"
    | "danger";
}
```

### children
- **Type:** **ReactNode**
- **Required:** Yes
- Button content

### palette
- **Type:** **ColorPaletteType**
- **Default:** **undefined**
- Color palette for styling (**"default"** | **"dark"** | **"monochrome"** | **"softBeach"**)

### variant
- **Type:** **string**
- **Default:** **"default"**
- Button style variant

## CSS Custom Properties

When using a palette, the component sets these CSS variables:

- **--button-hover-background-color**
- **--button-active-background-color**
- **--button-disabled-background-color**
- **--button-disabled-text-color**
- **--button-hover-box-shadow**
- **--button-focus-ring-color**
- **--button-backdrop-background-color**

## Accessibility

The Button component:
- Uses semantic **\<button>** element
- Supports all standard button attributes
- Defaults to **type="button"** to prevent form submission
- Respects disabled state
- Supports keyboard navigation
