## Import

```tsx
import { Input } from "@enotion/components";
```

```tsx
<Input placeholder="Enter text" />
```

```tsx
  palette="dark"

```

## Props

```typescript
interface InputProps extends InputHTMLAttributes<HTMLInputElement> {
  palette?: ColorPaletteType;
}
```

### palette
- **Type:** **ColorPaletteType**
- **Default:** **undefined**
- Color palette for styling

All standard HTML input attributes are supported.

```tsx
{/* Search */}
```

## CSS Custom Properties

When using a palette, the component sets:

- **--input-border-color** - Default border color
- **--input-focus-border-color** - Border color on focus
- **--input-background-color** - Background color
- **--input-text-color** - Text color
- **--input-placeholder-color** - Placeholder text color (60% opacity)
- **--input-disabled-background-color** - Background when disabled (10% opacity)

### Controlled Input

```tsx
import { useState } from "react";

function MyForm() {
  const [value, setValue] = useState("");

  return (
    <Input
      value={value}
      onChange={(e) => setValue(e.target.value)}
      placeholder="Controlled input"
      palette="softBeach"
    />
  );
}
```

### With Validation

```tsx
import { useState } from "react";

function ValidatedInput() {
  const [email, setEmail] = useState("");
  const [error, setError] = useState("");

  const validate = (value: string) => {
    if (!value.includes("@")) {
      setError("Invalid email");
    } else {
      setError("");
    }
  };

  return (
    <div>
      <Input
        type="email"
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          validate(e.target.value);
        }}
        palette="dark"
        style={{ borderColor: error ? 'red' : undefined }}
      />
      {error && <span style={{ color: 'red' }}>{error}</span>}
    </div>
  );
}
```

## Accessibility

The Input component:
- Supports all standard HTML input attributes
- Works with labels via **id** and **htmlFor**
- Supports **required**, **disabled**, **readonly** attributes
- Respects **aria-*** attributes for enhanced accessibility
- Provides proper color contrast with palette integration

### Accessibility Example

```tsx
<div>
  <label htmlFor="accessible-input">
    Email Address
    <span aria-label="required">*</span>
  </label>
  <Input
    id="accessible-input"
    type="email"
    required
    aria-required="true"
    aria-describedby="email-hint"
    palette="dark"
  />
  <span id="email-hint">We'll never share your email.</span>
</div>
```
